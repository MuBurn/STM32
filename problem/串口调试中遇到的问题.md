## 1. 串口重定向（半主机模式）

**半主机模式** 是 ARM 提供的一种机制，允许嵌入式设备上的代码使用主机的 I/O 资源（如键盘、屏幕、文件系统）。

### 工作原理：

text

```
嵌入式设备 <---> 调试器（ST-Link/J-Link） <---> PC主机
     ↑                    ↑                      ↑
   printf()         半主机协议          显示在调试器窗口
```



当你在代码中调用 `printf()` 时：

1. **默认情况下**，`printf` 会通过半主机模式，将数据发送给调试器
2. **调试器** 再将数据显示在 IDE 的控制台或调试窗口
3. **串口（UART）并没有被使用！**

#### **问题所在**

在你的代码中：

c

```
int fputc(int ch, FILE *f)
{
  HAL_UART_Transmit(&huart1, (uint8_t*)&ch, 1, 100);
  return ch;
}
```



**这个重定向只有在禁用半主机模式后才会生效！**

### 重定向流程对比：

**禁用半主机前：**

text

```
printf("Hello") → 半主机调用 → 调试器 → IDE调试窗口
                                      ↑
                                （串口没有数据！）
```



**禁用半主机后：**

text

```
printf("Hello") → fputc → UART发送 → 串口工具显示
```

## 问题3：不支持浮点数 

默认情况下，`arm-none-eabi-gcc` 的 `newlib-nano` 库不仅禁用 `printf` 浮点输出，也禁用 `scanf` 浮点输入

GCC 嵌入式标准库默认禁用浮点格式化以节省 Flash

| 配置                     | Flash 增量 | RAM 增量 | 说明                    |
| ------------------------ | ---------- | -------- | ----------------------- |
| 仅 `-u _printf_float`    | ~3~5KB     | ~100B    | 仅支持浮点输出          |
| 同时加 `-u _scanf_float` | ~6~8KB     | ~200B    | 同时支持浮点输入 + 输出 |

如果要格式化输入输出**浮点数**（`printf/%f`、`scanf/%f`），或者启用 `scanf`/`fscanf` 等函数对浮点数（`%f`/`%lf`）的解析能力
